# .impeccable.md

Design context for **Shopsugi** — persistent guidelines for all design work in this project.

---

## Design Context

### Users

Primary audience: women 28-45, interior decoration enthusiasts and curators who seek unique handcrafted pieces for their homes. They value authorial design, natural materials, and storytelling behind each piece. They browse with intention, often comparing pieces visually, and are willing to pay a premium for authenticity. Secondary use case: gift-giving for people who appreciate artisanal quality.

### Brand Personality

**Sofisticada, curada, poetica** (Sophisticated, curated, poetic)

- Voice: warm but refined. Speaks like a gallery curator who genuinely loves what they show — never salesy, never cold.
- Tone: contemplative, sensorial. Descriptions evoke texture, light, and the hands that made each piece.
- Emotional goal: the interface should feel like entering a quiet, well-lit atelier — a sense of calm discovery and trust.
- The brand name references **Kintsugi** — the Japanese art of repairing broken pottery with gold. This philosophy (beauty in imperfection, value in repair) is the spiritual core of all design decisions.

### Aesthetic Direction

**Visual tone**: Editorial warmth. Clean layouts with generous whitespace, but never sterile. The gold palette adds richness without excess.

**References**: Etsy / Elo7 — marketplace feel with artisanal curadoria, organic visual language, emphasis on product photography and maker stories.

**Anti-references**: Loud e-commerce (Shopee, AliExpress); overly techy/dark SaaS dashboards; neon colors; aggressive CTAs with countdown timers; anything that feels mass-produced or disposable.

**Theme**: Light mode only. Warm cream/parchment backgrounds (`warm-50`, `cream`) with kintsugi gold as the primary accent. Dark warm tones (`warm-800/900`) reserved for hero sections, footer, and admin sidebar.

### Typography

| Role | Font | Usage |
|------|------|-------|
| Display | Playfair Display | Headings, hero text, section titles — `font-display` |
| Body | DM Sans | All body text, UI labels, buttons, nav — `font-sans` |

Hierarchy relies on weight contrast (400-800) and size steps, not color variety. Headings use `font-extrabold`, body uses `font-medium` / `font-semibold`.

### Component Patterns

- **Cards**: `rounded-2xl`, white bg, `border-warm-200`, `shadow-sm` resting / `shadow-xl` hover with `-translate-y-1`
- **Buttons (primary)**: `rounded-full` or `rounded-xl`, `bg-kintsugi-500`, white text, `hover:bg-kintsugi-600`, `active:scale-95`
- **Buttons (secondary)**: `rounded-full`, border with warm or white tones, transparent/translucent bg
- **Inputs**: `rounded-xl`, `border-warm-200`, `bg-warm-50`, `focus:ring-2 focus:ring-kintsugi-500`
- **Pills/Badges**: `rounded-full`, small text, used for tags (`bg-kintsugi-50 text-kintsugi-500`) and categories (`bg-white/80 backdrop-blur-sm`)
- **Icons**: Inline SVGs, `strokeWidth={1.5-1.8}`, no icon library
- **Transitions**: `duration-150` for color changes, `duration-200-300` for transforms, `duration-500` for image scale on hover
- **Layout max-width**: `max-w-7xl` with `px-4 sm:px-6 lg:px-8`

### Design Principles

1. **Warmth over sterility** — Every surface should feel tactile and inviting. Prefer warm neutrals over pure grays. White is used for cards and elevated surfaces, not flat backgrounds.

2. **Quiet confidence** — The interface should guide without shouting. No aggressive animations, no neon highlights, no urgency tactics. Let the products and their craft speak.

3. **Gold as punctuation** — Kintsugi gold (#D4A017) is the signature accent. Use it deliberately for interactive elements, highlights, and moments of emphasis — never as a fill color for large surfaces.

4. **Generous space, intentional rhythm** — Sections breathe with `py-14`+ spacing. Cards use consistent padding. The eye should flow naturally from heading to content to action.

5. **Accessibility is care** — WCAG AA contrast compliance. Semantic HTML (`aria-label`, `role`, landmarks). Keyboard-navigable interactions. Respecting `prefers-reduced-motion`. This aligns with the Kintsugi ethos: careful attention to every detail.

### Accessibility Requirements

- WCAG 2.1 AA compliance for all public-facing and admin pages
- All interactive elements must be keyboard accessible
- Form inputs must have visible labels and `aria-describedby` for error states
- Images must have meaningful `alt` text (product names, not "image")
- Color alone must not convey meaning (stock status uses text + color)
- Focus indicators must be visible (`focus:ring-2 focus:ring-kintsugi-500`)

### Tech Stack (Frontend)

- Tailwind CSS v4 (with `@theme` custom tokens in `resources/css/app.css`)
- React 19 + TypeScript
- Inertia.js v2 (no client-side router — server-driven page transitions)
- react-hot-toast for notifications
- react-hook-form + zod for form validation
- No component library (custom components throughout)
- No icon library (inline SVGs)
